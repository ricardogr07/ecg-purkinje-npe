# Contract-B mock bundle

This directory is the Day-2 mock the Design track builds against, so the demo
never waits on real inference. It is regenerated by `node generate.mjs`
(deterministic, seed 1234). The real Contract-B artifact replaces `results.json`
and `geometry.json` wholesale at the Day 4 to 5 sync; every UI component treats
optional blocks as possibly-missing, so a leaner real artifact degrades
gracefully rather than crashing.

These are MOCK numbers, chosen to be physiologically plausible and consistent
with the Day-2 identifiability snapshot (delta_iv tightest, branch_angle
loosest, a deliberate cv to init_length_lv ridge). They are NOT measured results.

## Files

- `results.json` a faithful Contract-B artifact (see `../../docs/contracts.md`).
- `geometry.json` the biventricular surface for the 3D view.
- `contract_b.schema.json` a JSON Schema for `results.json`, including the
  `calibration` extension the real artifact is expected to carry.
- `generate.mjs` the deterministic generator.

## What the mock ADDS beyond the Contract-B draft (please carry these in the real artifact)

The `docs/contracts.md` Contract-B draft does not yet name these; the demo needs
them and mocks reasonable values. Requesting they be added to the real
serializer:

1. `prior` (per-param `[lo, hi]`): needed to draw corner-plot axes and the
   prior-vs-posterior width bars without the UI hardcoding Contract-A ranges.
2. `reference_theta` (synthetic truth, per param): the ground-truth markers in
   the corner plot and the pinned/unknowable bars. Central to the honest
   "synthetic-truth" framing.
3. `param_meta` (per param: `alias`, `unit`, `block`, `label`): so UI copy uses
   the artifact's own labels/units instead of inventing them.
4. `calibration` block:
   - `coverage_curve { nominal[], before[], after[] }` (TARP, before/after conformal),
   - `sbc { <param>: { before[], after[] } }` (rank-histogram counts),
   - `sbc_ks_pvalue { before, after }`, `tarp_atc { before, after }`,
   - `conformal_t { <param>: number }` (per-param inflation factor).
5. `posterior_predictive_ecg.band_lo` / `band_hi` as full `[lead][sample]`
   matrices (the draft lists them but leaves shape implicit; the demo expects the
   same shape as `signal`).
6. `synthetic_truth: true` flag, so the UI shows the honest framing chip.
7. `activation_map.values` length MUST equal `geometry.vertices` length (indexed
   1:1). If the real mesh is a volume, provide a surface decimation for the view.

## Known gaps / nice-to-have (not blocking)

- No per-sample activation fields, so the wavefront animation sweeps the single
  MAP/reference LAT field over time; it does not re-simulate as parameters move.
  If the real run can attach a few posterior-sample LAT fields, the map could
  morph across the posterior. Not required for the headline.
- ECG is a single synthetic beat; a real multi-beat strip would also render.
